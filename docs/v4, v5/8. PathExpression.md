FXP will support [path-expression-matcher](https://github.com/NaturalIntelligence/path-expression-matcher) for powerful pattern matching capabilities while maintaining 100% backward compatibility from v5.5.0.

## New Features

### 1. Advanced Pattern Matching in stopNodes

**Before (v4):**
```javascript
stopNodes: ["*.script", "*.style"]  // Limited to wildcard prefix
```

**After (v5 with path-expression-matcher):**
```javascript
import { Expression } from 'path-expression-matcher';

stopNodes: [
  "..script",                           // Deep wildcard - script anywhere
  "..style",                            // Deep wildcard - style anywhere
  new Expression("html.body.script"),   // Exact path
  new Expression("..pre"),              // Any pre tag
  new Expression("div[class=code]"),    // With attribute condition
  new Expression("item:first"),         // Position selector
  new Expression("ns::tag")             // Namespace support
]
```

### 2. Matcher-Based Callbacks (jPath Option)

**New Option: `jPath`**
- **Type:** `boolean`
- **Default:** `true` (backward compatible)
- **Description:** Controls whether callbacks receive jPath string or Matcher instance

#### With jPath: true (Default - Backward Compatible)

```javascript
const parser = new XMLParser({
  jPath: true,  // Default - callbacks receive string
  tagValueProcessor: (tagName, val, jpath, hasAttrs, isLeaf) => {
    if (jpath === "root.users.user") {
      return val.toUpperCase();
    }
    return val;
  }
});
```

#### With jPath: false (New - Matcher Instance)

```javascript
import { Expression } from 'path-expression-matcher';

const userExpr = new Expression("..user[type=admin]");

const parser = new XMLParser({
  jPath: false,  // Callbacks receive Matcher instance
  tagValueProcessor: (tagName, val, matcher, hasAttrs, isLeaf) => {
    if (matcher.matches(userExpr)) {
      return enhanceAdminUser(val);
    }
    return val;
  }
});
```

### 3. Pattern Syntax

#### Basic Patterns
```javascript
"root.users.user"           // Exact path (3 levels)
"*.users.user"              // Wildcard: exactly 2 levels, any first tag
"root.*.user"               // Wildcard: exactly 3 levels, any middle tag
```

#### Deep Wildcard
```javascript
"..user"                    // user anywhere (any depth)
"root..user"                // user anywhere under root
"..users..user"             // users somewhere, then user below it
```

#### Attribute Matching
```javascript
"user[id]"                  // user with "id" attribute
"user[type=admin]"          // user with type="admin"
"..user[status=active]"     // Active user anywhere
```

#### Position Selectors
```javascript
"item:first"                // First item (counter=0)
"item:nth(2)"               // Third item (counter=2, zero-based)
"item:odd"                  // Odd-numbered items
"item:even"                 // Even-numbered items
```

#### Namespace Support
```javascript
"soap::Envelope"            // Envelope with namespace "soap"
"ns::user[id]"              // Namespaced user with id attribute
"*::item"                   // item with any namespace
"..ns::tag"                 // Namespaced tag anywhere
```

## Backward Compatibility

### Automatic Pattern Conversion

For 100% backward compatibility, the old wildcard syntax is automatically converted:

**Old Syntax (v4):**
```javascript
stopNodes: ["*.script", "*.style"]
```

**Automatically Converted To:**
```javascript
stopNodes: ["..script", "..style"]
```

This happens transparently in `OptionsBuilder.js`:

```javascript
if (built.stopNodes && Array.isArray(built.stopNodes)) {
  built.stopNodes = built.stopNodes.map(node => {
    if (typeof node === 'string' && node.startsWith('*.')) {
      // Old syntax: *.tagname meant "tagname anywhere"
      // Convert to new syntax: ..tagname
      return '..' + node.substring(2);
    }
    return node;
  });
}
```

⚠ **warning:** Due to backward compatibility, when you use string in stopNodes or any configuration to specify a path, eg. `"*.script"` it matches to any level (i.e. `"..script"`). But when you use the same expression with Expression object `new Expression("*.script")` it will match to one level only. So it is recommended to use `"..script"` instead of `"*.script"` when you want to match to any level. Or shift to new approach.

### Breaking Changes

**NONE.** All existing code continues to work:
- ✅ Old `"*.tag"` patterns auto-converted
- ✅ Default `jPath: true` maintains string-based callbacks
- ✅ All existing options work as before

## Affected Callbacks

```typescript

  tagName: string,
  tagValue: string,
  jPath: string,
  hasAttributes: boolean,
  isLeafNode: boolean

// With jPath: false
tagValueProcessor: (
  tagName: string,
  tagValue: string,
  matcher: Matcher,
  hasAttributes: boolean,
  isLeafNode: boolean

```

### 2. attributeValueProcessor
```typescript
// With jPath: true (default)
attributeValueProcessor: (

// With jPath: false
```


## Migration Guide

### Upgrading from v4 to v5

#### No Changes Required
Your existing code will continue to work without any modifications.

#### Optional: Use New Features

**Before:**
```javascript
const parser = new XMLParser({
  stopNodes: ["*.script", "*.style"]
});
```

**After (using new patterns):**
```javascript
import { Expression } from 'path-expression-matcher';

const parser = new XMLParser({
  stopNodes: [
    new Expression("..script"),
    new Expression("..style"),
    new Expression("html.body.div[class=code]")  // More specific
  ]
});
```

**After (using Matcher in callbacks):**
```javascript
import { Expression } from 'path-expression-matcher';

const adminExpr = new Expression("..user[type=admin]");

const parser = new XMLParser({
  jPath: false,
  tagValueProcessor: (tagName, val, matcher) => {
    if (matcher.matches(adminExpr)) {
      return enhanceValue(val);
    }
    return val;
  }
});
```

## Performance

### Expression Pre-compilation

**✅ GOOD - Parse once, reuse many times:**
```javascript
const expr = new Expression("..user[id]");

const parser = new XMLParser({
  stopNodes: [expr],
  jPath: false,
  tagValueProcessor: (tagName, val, matcher) => {
    if (matcher.matches(expr)) {
      // Fast matching - expression already parsed
    }
    return val;
  }
});
```

**❌ BAD - Parse on every callback:**
```javascript
const parser = new XMLParser({
  jPath: false,
  tagValueProcessor: (tagName, val, matcher) => {
    // Slow - creates new Expression every time
    if (matcher.matches(new Expression("..user[id]"))) {
      // ...
    }
    return val;
  }
});
```


## Examples

### Example 1: HTML Parsing with Stop Nodes

```javascript
import { XMLParser } from 'fast-xml-parser';

const options = {
  ignoreAttributes: false,
  preserveOrder: true,
  stopNodes: ["..script", "..style", "..pre"],  // Auto-converted from old *.tag syntax
  unpairedTags: ["hr", "br", "link", "meta"]
};

const parser = new XMLParser(options);
const result = parser.parse(htmlContent);
```

### Example 2: Conditional Processing with Matcher

```javascript
import { XMLParser } from 'fast-xml-parser';
import { Expression } from 'path-expression-matcher';

const adminExpr = new Expression("..user[type=admin]");
const firstItemExpr = new Expression("..item:first");

const parser = new XMLParser({
  jPath: false,
  tagValueProcessor: (tagName, val, matcher) => {
    if (matcher.matches(adminExpr)) {
      return { ...val, enhanced: true };
    }
    if (matcher.matches(firstItemExpr)) {
      return { ...val, isFirst: true };
    }
    return val;
  }
});
```

### Example 3: Namespace-Aware Processing

```javascript
import { XMLParser } from 'fast-xml-parser';
import { Expression } from 'path-expression-matcher';

const soapBodyExpr = new Expression("soap::Envelope.soap::Body");

const parser = new XMLParser({
  jPath: false,
  removeNSPrefix: false,  // Keep namespaces
  updateTag: (tagName, matcher, attrs) => {
    if (matcher.matches(soapBodyExpr)) {
      console.log("Processing SOAP body");
    }
    return tagName;
  }
});
```
